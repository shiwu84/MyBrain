这篇笔记主要介绍了 Nushell 的**安装**和**设置**。

### 1. 官方文档

笔记首先提供了一个指向 Nushell **官方文档**的链接：[https://www.nushell.sh/zh-CN/book/](https://www.nushell.sh/zh-CN/book/)。这意味着如果你想深入了解 Nushell 的功能和用法，可以访问这个链接。

### 2. 安装

笔记详细说明了在不同操作系统上安装 Nushell 的方法：

-   **Arch Linux**:
    -   可以直接通过 `pacman -S nushell` 命令使用包管理器进行安装。这表明 Nushell 在 Arch Linux 的包管理生态中是可用的。
-   **Windows 11**:
    -   可以使用 `winget install nushell` 命令进行安装。这里特别提到是“用户范围内安装”，意味着它不会影响系统级别的默认 shell 设置。

### 3. 在 Kitty 中设置默认 Shell

如果你使用 Kitty 终端模拟器，笔记提供了将其默认 shell 设置为 Nushell 的步骤：

-   按 `Ctrl+Shift+F2` 打开 `kitty.conf` 配置文件。
-   找到 `shell` 变量所在的行，取消注释（如果被注释了）。
-   将该行中的路径替换为 Nushell 的安装路径。

### 4. 将 Nu 设置为登录 Shell

笔记还说明了如何将 Nushell 设置为系统的登录 shell：

-   使用命令 `chsh -s $(which nu)`。这个命令会更改你的用户默认 shell 为 Nushell 的可执行文件路径。

-   **重要提示**: 笔记中有一个重要的提示框：
    > [!NOTE]
    > 这在某些情况下可能会出现问题，因为Nu并不兼容POSIX。

    这意味着虽然可以将 Nushell 设置为登录 shell，但由于 Nushell 不完全兼容 POSIX 标准（大多数 Linux/Unix 系统默认的 shell 行为标准），这可能会导致一些脚本或系统功能无法正常工作。因此，在将其设置为登录 shell时需要谨慎。
